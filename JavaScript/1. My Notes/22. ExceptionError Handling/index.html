Definition

Exception handling in JavaScript involves the use of try-catch-finally blocks to manage errors that may occur during the execution of code. This mechanism allows you to gracefully handle and recover from errors without causing the script to stop abruptly.

Syntax:

try {
  // code that may throw an error
} catch (err) {
  // code to be executed if an error occurs
} finally {
  // code to be executed regardless of an error occurs or not
}

Exception Handling Statements

JavaScript provides an error-handling mechanism to catch runtime errors using try-catch-finally block.

An exception is thrown whenever an error occurs. An exception can be a string, number, Boolean, or an Object.

  • try block: Encloses the code that might throw an exception. If an error occurs within this block, it gets "caught" and triggers the catch block.
  • catch block: Catches and handles any error that occurs within the corresponding try block. It specifies how to handle the caught error. The error variable holds information about the error.
  • finally block: It's an optional block that follows a try...catch block. The code within the finally block executes irrespective of whether an error occurs or not.
  • throw statement: allows us to throw our custom exceptions. When throw statement is executed, the statements present after it will not execute. The control will directly pass to the catch block.

Example

<script>
document.getElementById('Exception Handling').addEventListener('click', () => {
	const age = document.getElementById('age').value;
	try{
		if(isNaN(age) || age === ''){
    document.getElementById('output-1').innerHTML = 'Please enter your age.';
		}else if(age < 18){
			throw new Error('Not Eligible');
		}else{
			document.getElementById('output-1').innerHTML = 'You are Eligible for voting.'
		}
	} catch (error){
		document.getElementById('output-1').innerHTML = error;
	} finally{
		document.getElementById('output-2').innerHTML = `Your Age: ${age}`;
	}
});
</script>

Types of Error

While coding, there can be three types of errors in the code:

  • Syntax Error: When a user makes a mistake in the pre-defined syntax of a programming language, a syntax error may appear.
  • Runtime Error: When an error occurs during the execution of the program, such an error is known as Runtime error. The codes which create runtime errors are known as Exceptions. Thus, exception handlers are used for handling runtime errors.
  • Logical Error:An error which occurs when there is any logical mistake in the program that may not produce the desired output, and may terminate abnormally. Such an error is known as Logical error.

Error Objects

Error objects are thrown when runtime errors occur. The following are the different type of Error.

Error Name Error Description
RangeError Error that occurs when a numeric variable or parameter is outside of its valid range.
ReferenceError Error that occurs when de-referencing an invalid reference, i.e; A ReferenceError is thrown if we use a variable that has not been declared.
SyntaxError Error that occurs while parsing code in eval() method.
TypeError Error that occurs when a variable or parameter is not of a valid type.
URIError Error that occurs when encoded URI() or decodeURI() are passed invalid parameters. (Uniform Resource Identifier)

All mentioned objects above contain the following properties:

Peoperty Description
name Returns the error name
message Returns the error message.

You cannot set the number of significant digits of a number to 500:

    let num = 1;

    // Trying to set the number of significant digits of 'num' to 500
