JavaScript Number

In javascript, the Number is a primitive data type. It represents all numerical values like:

  • Integer: 2, 5, 18, 100, 55, etc
  • Float: 1.0, 2.5, 8.4503, 245.50, etc
  • Exponential: .23e5, 1.23e-5, etc
  • Binary: 0b1010, 0b1111, etc
  • Octal: 0o7, 0o77, 0o777, etc
  • Hexadecimal: 0xFF, 0xFFF, 0xFFFF, etc

JavaScript numbers are always stored as double precision floating point numbers. This format stores numbers in 64 bits, where the number (the fraction) is stored in bits 0 to 51, the exponent in bits 52 to 62, and the sign in bit 63:

NOTE:-

  • Number can only keep precision of 17 digits.

Ways to define numbers in javaScript

There are two ways to define Numbers in javascript

  • Number literals
  • Number Constructor

1. Number literals

In JavaScript, number literals are representations of numerical values directly within the code. They can be written as integers, floating-point numbers, or using special notations like exponential form.

Example

<script>
    //Integer Literals:
    let integerLiteral = 42; // Integer number literal
    let negativeLiteral = -10; // Negative integer literal
    let binaryLiteral = 0b1010; // Binary number literal (0b prefix)
    let octalLiteral = 0o777; // Octal number literal (0o prefix)
    let hexadecimalLiteral = 0xFF; // Hexadecimal number literal (0x prefix)

    //Floating-Point Literals:
    let floatLiteral = 3.14; // Floating-point number literal
    let anotherFloat = -0.01; // Negative floating-point number literal
    let exponentialLiteral = 6.022e23; // Scientific notation (exponential form)

    //Special Values:
    let infinity = Infinity; // Represents infinity
    let negInfinity = -Infinity; // Represents negative infinity
    let notANumber = NaN; // Represents "Not-a-Number" value (results from invalid operations)

    //Exponential Notation:
    let bigNumber = 1.5e6; // 1.5 * 10^6 (1.5 million)
    let smallNumber = 2.5e-3; // 2.5 * 10^-3 (0.0025)

    document.getElementById('define-output-1').innerHTML =
        `integerLiteral = 42;    DateType: ${typeof integerLiteral}<br>
negativeLiteral = -10;    DateType: ${typeof negativeLiteral}<br>
binaryLiteral = 0b1010;    DateType: ${typeof binaryLiteral}<br>
octalLiteral = 0o777;    DateType: ${typeof octalLiteral}<br>
hexadecimalLiteral = 0xFF; ${typeof hexadecimalLiteral}<br>
floatLiteral = 3.14;    DateType: ${typeof floatLiteral}<br>
anotherFloat = -0.01;    DateType: ${typeof anotherFloat}<br>

smallNumber = 2.5e-3;    DateType: ${typeof smallNumber}`;
</script>

2. Number Constructor

In JavaScript, the Number constructor is a built-in function used to create number objects. It can be used with or without the new keyword to create number objects from numeric values or strings.

Example

<script>
    // Creating a number object from a numeric value
    let num1 = Number(42);

    // Creating a number object from a string
    let num2 = Number("3.14");

    // Trying to create a number object from an invalid string
    let invalidNum = Number("hello");

    // Creating a number object using the new keyword
    let numObj1 = new Number(10);

    // Creating a number object from a string using new
    let numObj2 = new Number("99");

    document.getElementById('Constructor-output-1').innerHTML =
        `num1 = Number(42); ${typeof num1}<br>
num2 = Number("3.14"); ${typeof num2}<br>
invalidNum = Number("hello"); ${typeof invalidNum}<br>
numObj1 = new Number(10); ${typeof numObj1}<br>
numObj2 = new Number("99"); ${typeof numObj2}<br>`;
</script>

Javascript Number Properties

In JavaScript, the Number object has several properties that provideinformation about numbers and mathematical constants.

  • Number.MAX_VALUE: Represents the maximum numeric value that can be represented in JavaScript.
  • Number.MIN_VALUE: Represents the smallest positive numeric value greater than zero that can be represented in JavaScript.
  • Number.NaN: Represents a value that is "Not-a-Number."
  • Number.NEGATIVE_INFINITY: Represents negative infinity, returned on overflow.
  • Number.POSITIVE_INFINITY:Represents positive infinity, returned on overflow.
  • Number.MIN_SAFE_INTEGER: Minimum safe integer in javascript is -(253 - 1)
  • Number.MAX_SAFE_INTEGER: Maximum safe integer in javascript is 253 - 1
  • Number.EPSILON: Represents the difference between 1 and the smallest floating-point number greater than 1.

Example

Javascript Number methods

JavaScript's Number object comes with several built-in methods that facilitate working with numbers, performing operations, and accessing specific numerical information.

  • Number.isNaN(): Checks if a value is NaN.
  • Number.isFinite(): Checks if a value is a finite number.
  • Number.isInteger(): Checks if a value is an integer.
  • Number.isSafeInteger(): Checks if a value is a safe integer within a specified range.
  • Number.parseFloat(): Parses a string argument and returns a floating-point number.
  • Number.parseInt(): Parses a string argument and returns an integer.
  • Number.toExponential(): Returns a string representing the number in exponential notation.
  • Number.toFixed(): Returns a string representing the number in fixed-point notation with a specified number of digits.
  • Number.toLocaleString(): Returns a string representing the number using the local language format.
  • Number.toPrecision(): Returns a string representing the number to a specified precision in fixed or exponential notation.
  • Number.toString(): Returns a string representing the specified number object.
  • Number.valueOf(): Returns the primitive value of a Number object.

Example

<script>
   let myNumber = 24.12345;
    let parsedFloat = Number.parseFloat("3.14 meters"); // Parse a float from a string
    let parsedInt = Number.parseInt("10");  // Parse an integer from a string

    let numValue = myNumber.valueOf(); // Get the primitive value of the number
